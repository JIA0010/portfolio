About Me

CHEN JIAです。42Tokyoと放送大学に在籍しつつ、現役のWebエンジニアとして活動しています。

約2年間、個人事業主としてCtoCプラットフォーム開発や業務委託、自習室運営などに挑戦しました。この経験から、技術的な視点だけでなく、ビジネスサイドの課題解決にも強い関心を持っています。

現在は、TypeScriptを用いた開発、特にReact (Next.js)やNestJS、そしてクリーンアーキテクチャやTDD（テスト駆動開発）といった、スケーラブルで保守性の高いシステム設計技術の習得に注力しています。

42TokyoのCommon Coreを突破し、Advancedを履修するためシンガポールへのトランスファー準備中です。グローバルな開発環境での学びを目指しています。

Featured Project

Featured

SaboLock (サボロック)

GPS位置情報と金銭的コミットメントを組み合わせた、行動経済学ベースの習慣化支援Webアプリケーション

SaboLockは、リモート学習やフリーランスワークにおける自己管理の難しさを解決するために開発したWebアプリケーションです。「デポジット + GPS連動判定」により、行動せざるを得ない環境を創出し、習慣化を支援します。

採用技術・アーキテクチャ

  • Frontend: React, Next.js 15, TypeScript
  • Backend: NestJS 11, TypeScript, TypeORM
  • Database: PostgreSQL 17
  • Infrastructure: Docker, Vercel, Heroku
  • Architecture: レイヤードアーキテクチャ、Monorepo (pnpm Workspaces)
  • Integration: Stripe API, Google Maps API, Nodemailer

担当領域

フルスタック開発。要件定義からDB設計、API開発、フロントエンド実装、インフラ構築まで一貫して担当しました。企画書の作成から、ER図・シーケンス図の設計、Docker環境構築、CI/CD構築まで、プロダクト開発の全工程を経験しました。

工夫した点

  • モノレポアーキテクチャ: pnpm Workspacesで型定義やユーティリティを共有し、フロントエンド・バックエンド間の開発効率を向上
  • クリーンアーキテクチャ: NestJSのDIコンテナとモジュール分割により、疎結合で保守性の高い設計を実現
  • 外部API統合: Stripe決済、Google Maps API、メール送信など、複数の外部サービスを統合し、適切なエラーハンドリングとリトライ処理を実装
  • セキュリティ: JWT + bcryptによる認証基盤、TypeORMでのマイグレーション管理
  • 開発環境: Docker Composeによる環境統一とGitHub ActionsによるCI/CD構築
